# 部署与回滚说明（Win Server + MySQL）

> 推荐优先使用新版内网部署脚本与指南：
> - `ops/WinServer内网部署指南.md`
> - `ops/winserver/deploy_winserver.ps1`
> - `ops/winserver/deploy_from_bundle.ps1`

## 1. 后端部署

```powershell
cd D:\病种分析V2
powershell -ExecutionPolicy Bypass -File .\ops\deploy_backend.ps1 -ProjectRoot "D:\病种分析V2"
```

启动命令（建议挂到计划任务或 NSSM）：

```powershell
cd D:\病种分析V2\backend
.\.venv312\Scripts\python.exe -m uvicorn app.main:app --host 0.0.0.0 --port 18080
```

## 2. 前端部署

```powershell
cd D:\病种分析V2
powershell -ExecutionPolicy Bypass -File .\ops\deploy_frontend.ps1 -ProjectRoot "D:\病种分析V2" -DistTarget "D:\deploy\disease-analytics-frontend"
```

将 `D:\deploy\disease-analytics-frontend` 配置到 IIS 站点根目录即可。

## 3. 数据库备份

```powershell
cd D:\病种分析V2
powershell -ExecutionPolicy Bypass -File .\ops\backup_db.ps1
```

## 4. 数据库回滚

```powershell
cd D:\病种分析V2
powershell -ExecutionPolicy Bypass -File .\ops\restore_db.ps1 -SqlFile "D:\backup\disease-analytics\disease_analytics_YYYYMMDD_HHMMSS.sql"
```

## 5. 发布门禁建议

1. 先执行数据库备份。
2. 执行后端部署脚本并重启服务。
3. 执行前端构建并替换静态目录。
4. 回归接口：`/api/v1/health`、`/api/v1/quality/overview`、`/api/v1/reports/executive`。
5. 异常时优先恢复数据库，再回滚前端静态包与后端代码目录。
