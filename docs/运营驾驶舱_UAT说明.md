# 运营驾驶舱 UAT 说明

## 目标

本轮 UAT 用于确认系统已经从“病种/DIP 分析工具”切换为“医院科室运营驾驶舱”，并验证管理员与科主任的权限边界、主页面信息结构和导出能力是否符合预期。

## 测试角色

- 管理员：
  - 可查看全院运营驾驶舱
  - 可查看任意科室分析
  - 可进入数据导入中心
  - 可导出全量运营病例报表
- 科主任：
  - 默认进入本科室分析
  - 可查看全院科室评分排名
  - 不可进入数据导入中心
  - 不可查看其他科室详情
  - 导出病例报表时只能导出本科室且需脱敏

## 核心场景

### 1. 管理员登录

预期：

- 默认进入“运营驾驶舱”
- 顶部菜单显示：
  - `运营驾驶舱`
  - `科室分析`
  - `报表中心`
  - `数据导入中心`
- 首页展示：
  - 出院人次
  - 综合运营均分
  - 平均住院日
  - 周转指数
  - 风险科室数
- 首页主区块为“科室综合运营评分排行榜”

### 2. 科主任登录

预期：

- 默认进入“科室分析”
- 顶部菜单不显示 `数据导入中心`
- 科室选择默认锁定为本科室
- 可以看到全院评分排名与科室名称
- 手工修改 URL 或请求参数时，无法打开其他科室详情

### 3. 数据导入中心

管理员验证：

- 仅保留运营分析相关导入：
  - 病案首页
  - 病案出院数据
  - 费用汇总数据
  - 费用明细数据
- 不再出现 DIP 字典、待映射池、人工回填等前台能力
- 可查看批次历史、问题明细、备份、清除和恢复

科主任验证：

- 无法从菜单进入
- 手工访问 `/imports` 会被重定向回允许页面

### 4. 科室分析页

验证点：

- 页面包含：
  - 综合评分
  - 出院人次
  - 次均费用
  - 平均住院日
  - 异常命中
- 页面展示：
  - 月度趋势
  - 费用结构
  - 医师对比
  - 评分驱动因素
  - 异常规则分布
  - 高金额项目 TOP
  - 建议动作
- 页面中不再出现 DIP 收益、DIP 结余、入组率等内容

### 5. 报表中心

验证点：

- 可查看全院月度趋势
- 可查看科室评分月报
- 点击“导出运营病例报表 CSV”时：
  - 管理员导出全量数据
  - 科主任导出本科室数据，姓名与住院号脱敏

## 权限回归建议

- 使用管理员账号访问任意科室详情，应成功
- 使用普外科主任账号访问普外科详情，应成功
- 使用普外科主任账号直接请求儿科详情，应返回拒绝或 403
- 使用科主任账号访问导入批次接口，应返回拒绝或无法进入页面

## 废弃能力确认

本轮 UAT 中，以下能力不再作为前台验收范围：

- DIP 统计
- 病种优先级筛选
- 漏洞分析前台页面
- DIP 版科主任专题

这些接口仍可能在后端保留，但仅用于过渡兼容，不属于当前产品主线。
