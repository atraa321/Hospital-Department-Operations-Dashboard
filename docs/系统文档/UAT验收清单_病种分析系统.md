## 一、UAT目标与范围

- 验证“导入 -> 分析 -> 预警 -> 工单 -> 闭环 -> 报表”的全链路可用性。
- 验证核心指标口径、权限控制、脱敏导出符合要求。
- 验证一期性能指标和稳定性目标满足上线门槛。

---

## 二、UAT入口准则

- 已部署UAT环境（数据库、缓存、对象存储、任务调度可用）。
- 已导入基础测试数据（病案、费用汇总、费用明细、DIP/ICD主数据）。
- 测试账号准备完成（管理员、科室主任、医务科、医保办、财务、查看用户）。
- 关键缺陷修复完毕：无Sev1，Sev2已关闭或有规避方案。
- 版本冻结：UAT期间仅允许缺陷修复，不新增需求。

---

## 三、测试数据准备清单

| 数据集ID | 说明 | 数据要求 |
|---|---|---|
| TD-001 | 正常病案数据 | 含完整住院号、入出院日期、诊断信息 |
| TD-002 | 异常病案数据 | 日期逆序、缺失住院号、非法编码 |
| TD-003 | 正常费用明细 | 覆盖24类费用类别 |
| TD-004 | 异常费用明细 | 非数值金额、负金额、重复记录、未识别类别 |
| TD-005 | 费用汇总 | 可与明细做对账 |
| TD-006 | DIP/ICD主数据 | 覆盖映射成功与失败场景 |

## 七、缺陷分级与处理SLA

| 级别 | 定义 | 响应时限 | 修复时限 |
|---|---|---|---|
| Sev1 | 阻塞上线/数据错误不可接受 | <= 2小时 | <= 24小时 |
| Sev2 | 核心功能受影响，有替代方案 | <= 4小时 | <= 3天 |
| Sev3 | 一般功能问题，不影响主链路 | <= 1天 | <= 7天 |
| Sev4 | 体验或建议优化 | <= 2天 | 版本排期 |
