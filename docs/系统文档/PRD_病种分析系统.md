## 一、项目概述

### 1.1 背景

医院面临支付方式改革（DIP/DRG）、医保穿透监管、成本与质量双重压力。当前业务痛点集中在：

- 数据分散，病案、费用、DIP主数据缺乏统一治理。
- 病种分析依赖人工经验，缺少统一口径与标准化流程。
- 问题发现后缺乏责任闭环，整改难持续。
- 病人来源分析未形成“分析-动作-转化”联动机制。

### 1.2 项目定位

建设“病种分析与运营管理平台”，形成以下闭环能力：

- 看得清：病种结构、费用结构、质量结构可视化。
- 控得住：异常识别、医保合规、规则预警可执行。
- 管得好：标准制定、责任分派、整改追踪、绩效联动。

### 1.3 项目目标

#### 1.3.1 业务目标（上线后 3-6 个月）

- 建立病种分析标准作业流程（SOP）并覆盖主要临床科室。
- 每月形成病种复盘机制，至少完成 1 个重点病种闭环整改。
- 药占比、次均费用、医保拒付等核心指标实现可监控、可追责。

#### 1.3.2 产品目标

- 支持多源Excel导入、规则校验、错误反馈、批次追踪。
- 支持病种画像、来源分析、费用/质量异常分析、闭环工单。
- 支持按角色的数据权限与脱敏展示。

#### 1.3.3 数据治理目标

- 住院号统一为跨表唯一关联主键。
- 构建“费用分类标准口径”并固化为规则。
- 建立数据质量看板：完整性、准确性、及时性、唯一性。

| DIP目录 | `基础数据/平顶山2025年DIP2.0分组目录库.xlsx` | 已具备 | DIP分组与分值主数据 |

### 4.2 导入顺序（强制）

1. 病案出院数据（建立病例主索引）。
2. 费用汇总数据（可选，仅更新总费用参考值）。
3. 费用明细数据（必做，按标准口径自动汇总覆盖分类费用）。
4. DIP与ICD主数据（按月/按政策更新）。

### 4.2.1 大批量费用明细导入限制（新增）

- 适用范围：单次导入全年费用清单或历史补录批次。
- 单文件大小上限：`<= 200MB`（系统参数可配置）。
- 单批次明细行上限：`<= 500,000` 行（系统参数可配置）。
- 单批次处理时长上限：`<= 20分钟`，超时自动终止并保留错误上下文。
- 导入并发限制：全局最多 `2` 个导入任务同时执行，超出进入队列。
- 超限策略：拒绝入库并返回明确错误码，提示按月拆分导入。

## 五、标准化业务SOP（核心）

### 5.1 总流程

`定病种 -> 拆结构 -> 找漏洞 -> 定标准 -> 强闭环`

每个步骤必须产出可审计结果，形成任务单与复盘记录。

### 5.2 Step1 定病种

#### 输入

- 病种出院人次、费用占比、盈亏波动、医保拒付、再入院率、平均住院日、路径变异率。

#### 量化评分模型

`DiseasePriorityScore = 0.25*人次得分 + 0.20*费用贡献得分 + 0.15*盈亏波动得分 + 0.15*拒付风险得分 + 0.10*住院日风险得分 + 0.10*再入院风险得分 + 0.05*路径变异得分`

- 各子项标准化到 0-100 分。
- 分层规则：
  - `>= 75`：主力病种
  - `60-74`：重点病种
  - `< 60`：常规监测病种

#### 输出

- 《病种分层名单（按科室）》
- 《月度重点病种清单》

### 5.3 Step2 拆结构

按病种对以下结构拆解：

- 临床结构：诊断、手术、并发症、路径节点。
- 费用结构：药、耗、检、治、手术、护理、服务、其他。
- 质量结构：住院日、再入院、不良事件、死亡。
- 来源结构：地域、渠道、转诊路径、时间分布。

输出：结构异常清单（高于基线、波动异常、占比突增）。

### 5.4 Step3 找漏洞（规则库驱动）

| 漏洞类型 | 典型规则 |
|---|---|
| 费用漏洞 | 药品未优选、耗材替代性差、辅助用药偏多、结构占比异常 |
| 诊疗漏洞 | 术前等待过长、路径执行偏离、住院日延长、适应症不匹配 |
| 医保合规漏洞 | 分解住院、重复收费、超适应症、项目串换、诊疗不一致 |

输出：规则命中记录 + 证据数据 + 建议动作。

### 5.5 Step4 定标准

| 标准类型 | 产物 |
|---|---|
| 深化版临床路径 | 路径节点、标准住院日区间、变异原因编码 |
| 医保管控清单 | 检查检验正负清单、药品限制清单、耗材优选目录 |
| 费用控制标准 | 药耗占比上限、次均费用红黄线 |
| 质量标准 | 再入院率、并发症率、死亡率等控制目标 |

每项标准必须具备：版本号、生效日期、适用科室、责任人、回滚版本。

### 5.6 Step5 强闭环

#### 工单流程

`规则命中 -> 自动建单 -> 指派责任人 -> 限期整改 -> 复核 -> 关闭/重开`

#### SLA

| 级别 | 首次响应 | 完成整改 |
|---|---|---|
| 红色 | <= 24小时 | <= 7天 |
| 橙色 | <= 48小时 | <= 14天 |
| 黄色 | <= 72小时 | <= 30天 |

#### 闭环考核

- 按期闭环率。
- 复发率（30天内同规则重复命中）。
- 规则命中下降幅度。

## 七、规则与口径定义

### 7.1 费用分类标准口径

| 原始费用类别 | 归并口径 |
|---|---|
| 西药费、成药费 | 药品费 |
| 草药费 | 草药费 |
| 材料费 | 材料费 |
| 检查费、CT费、心电图、彩超费、B超费、放射费、胃镜费、化验费 | 检查费 |
| 治疗费、理疗费、换药费、注射费、处置费 | 治疗费 |
| 手术费、麻醉费 | 手术费 |
| 护理费 | 护理费 |
| 床位费、诊查费 | 服务费 |
| 输氧费、输血费、未识别类别 | 其他费用 |

### 7.2 去重与冲销处理

- 明细唯一建议键：`住院号 + 收费项目编码 + 项目名称 + 单价 + 计费单位 + 收费科室 + 批次号 + 行哈希`。
- 不采用“覆盖更新”删除历史；采用“追加并标记来源批次”。
- 负金额视为冲销记录，参与净额计算并保留审计轨迹。

### 7.3 核心指标口径

| 指标 | 公式 |
|---|---|
| 次均费用 | 总费用 / 出院人次 |
| 药占比 | (药品费+草药费) / 总费用 |
| 耗材占比 | 材料费 / 总费用 |
| 平均住院日 | 总住院天数 / 出院人次 |
| 再入院率 | 31天再入院人次 / 出院人次 |
| 路径变异率 | 变异病例数 / 纳入路径病例数 |
| 拒付率 | 拒付病例数 / 结算病例数 |
| 闭环按期率 | 按SLA关闭工单数 / 总关闭工单数 |
| DIP模拟支付金额 | DIP分值 x 模拟点值 |
| DIP模拟结余 | DIP模拟支付金额 - 总费用 |
| 医师费用离散系数 | 同病种医师次均费用标准差 / 次均费用均值 |
| 耗材异常率 | 耗材异常病例数 / 病种总病例数 |
| 药品异常率 | 药品异常病例数 / 病种总病例数 |

### 7.4 预警分级规则

| 预警类型 | 黄色 | 橙色 | 红色 |
|---|---|---|---|
| 次均费用异常 | >科室均值1.3倍 | >1.6倍 | >2.0倍 |
| 药占比异常 | >45% | >50% | >60% |
| 平均住院日异常 | >科室均值1.5倍 | >1.8倍 | >2.0倍 |
| 拒付异常 | 月拒付率>基线+20% | >基线+40% | >基线+60% |
| 路径变异异常 | >20% | >30% | >40% |

### 7.5 DIP映射策略

1. 优先使用病案 `icd` 与DIP主诊断匹配。
2. 对 `诊断ICD` 先经ICD10映射到医保版后再匹配。
3. 手术编码先经ICD9映射到医保版，再参与DIP组合匹配。
4. 未命中记录入“待映射池”，支持人工映射回填并版本留痕。

### 7.6 科主任专题分析口径（新增）

1. TOP5病种范围：按“科室+统计周期”下主诊断出院人次降序取前5，若并列取费用贡献高者优先。
2. 医师对比范围：同病种下仅纳入病例数 `>= 5` 的医师，避免小样本偏差。
3. 异常判定口径：耗材/药品/检查费用占比超科室病种基线 `1.5` 倍或连续2期上升判定为异常。
4. DIP支付为模拟值：用于管理决策，不作为医保正式结算依据。

## 十三、风险与应对

| 风险 | 影响 | 应对 |
